About This Project

The ARNOVA Conference History Database is a structured dataset of presentation abstracts from the Association for Research on Nonprofit Organizations and Voluntary Action (ARNOVA) Annual Conference, spanning 2012 through 2025. It was constructed using AI-assisted PDF parsing to transform unstructured conference catalog PDFs into an analyzable research dataset.

This project is an independent research effort and is not affiliated with, endorsed by, or conducted on behalf of ARNOVA. All data were compiled exclusively from publicly available conference catalog PDFs accessible from the ARNOVA Conference Catalog page.

Inspiration: This project was inspired by the SSWR Conference History Database by Perron, Victor, & Qi (2026), which compiled 23,793 presentations from the Society for Social Work and Research conferences (2005–2026).

Methodology

Data Collection

Conference catalog PDFs were downloaded from the official ARNOVA website. The catalogs span 14 years (2012–2025) and contain a total of 5,591 presentation records.

PDF Parsing

The catalogs use six different PDF formats across the 14-year span, requiring format-specific parsers:

FormatYearsKey Characteristics
Compact multi-column2012–2015Landscape layout with title/abstract in left column, authors in right column. Parsed using word-coordinate extraction.
Single-column with email2016Portrait layout with email addresses on separate lines as entry anchors.
Email-in-parentheses2017–2020*Author(email) format without track/keyword metadata.
Inline track-keywords2021–2022[Track] – Keywords: inline format with semicolon-separated authors.
Standalone keywords2023Separate Keywords: line below track name.
Track + methodology2024Track: and Methodology Keywords: as labeled fields.

The 2025 catalog uses a schedule-based format containing presentation titles and authors but no abstracts.

Tools Used

  • pdfplumber — PDF text and word-coordinate extraction
  • openpyxl — Excel output generation
  • Python regex — Format detection and text parsing
  • Claude Code — AI-assisted parser development and debugging

Data Quality

The dataset was constructed through iterative development with manual verification at each stage:

  • Format detection: Each year's catalog was automatically classified into one of six formats based on structural markers (track labels, keyword patterns, email formats).
  • Entry counts: Parsed entry counts were compared against expected ranges based on catalog page counts and visual inspection.
  • Author parsing: Multiple delimiter strategies (semicolons, email-in-parentheses, line-based) were applied based on format to accurately separate co-authors.
  • Multi-column handling: Years 2012–2015 required word-coordinate-based column splitting (x=380 threshold) to properly separate title/abstract from author/affiliation columns.

Known Limitations

  1. Author disambiguation is not performed. The same researcher appearing with different name spellings or affiliations across years is counted as separate authors.
  2. Abstract coverage is 81%. The 2025 catalog is schedule-only (no abstracts), and some entries in earlier years may have truncated abstracts due to PDF layout complexity.
  3. Track/keyword metadata is available only for 2021–2024 (21% of entries). Earlier catalogs did not include this classification.
  4. Some title truncation may occur in the 2012–2015 multi-column format, where title continuation lines could be misclassified.
  5. The database captures published program content, not actual presentations delivered. Withdrawn presentations may be included.
  6. COVID-19 impact: The 2020–2021 virtual conferences show reduced presentation counts that reflect pandemic-era participation, not long-term trends.
  7. Affiliation data is point-in-time. Institutional affiliations reflect what was reported at each conference, not current positions.
